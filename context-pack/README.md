# OneTask / NerveCenter Codex Briefing Pack

Last updated: 2026-05-05

Purpose: let a new Codex session resume OneTask, NerveCenter, Shailos, and DeskPhone Web work with a small current briefing instead of rereading the codebase.

Start every session in this order:

1. Read this file.
2. Read `CURRENT_STATE.md`.
3. Read `NEXT_ACTIONS.md`.
4. Read `ARCHITECTURE_INDEX.md`.
5. Read `RECENT_CHANGES.md`.
6. Check live git status and latest commits.
7. Open only the exact files needed for the active task.

Keep these files current at the end of any meaningful session. Use `HANDOFF.md` for the broader living project document; use this pack for fast startup.
